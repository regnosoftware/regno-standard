Definition - ConversionDefDoc

The ConvDefinitionDoc document is a definition document that contains the properties for a unique conversion instance.

The ConvDefinitionDoc document does not contain a reference to the ConfigDoc document and therefore can be reused by multiple configurations and data sets.

Field Data Type Required Default Description
id String Yes - A unique identifier that is referenced by all other documents see section Unique Id Generation.
name String No - A free form string value to name the conversion.
conversionType Fixed String No Rational A fixed type description of the conversionType. Valid options are: Rational, Table, Text.
units String No - A free form string representing the conversion units.
format String No - A C style format used to display the conversion data value. See the section Format Strings for details and examples.
interpolate Fixed String No - A fixed string to enable interpolation of the mapped data values. Valid options are: True, False.
coefficients Array [64-bit Double] Yes - An array of 64-bit Double coefficients that are used for the rational conversion operation.
conversionTable Array [64-bit Double, 64-bit Double] Yes - An array of 64-bit Double key/pair values.
conversions Array [64-bit Double, String] Yes - An array of 64-bit Double and String key/pair values.
type Fixed String Fixed ConversionDefDoc The fixed value is always set to ConversionDefDoc.
